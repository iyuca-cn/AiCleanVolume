# 目录树会话级 JSON 缓存设计

## 背景

当前目录树的“懒加载”问题不在 UI，而在数据链路：

1. 首次扫描后，展开目录仍会反复调用 `folder-size-ranker-cli`。
2. 对 NTFS 子目录的 CLI 请求，本质上仍可能重新扫描整卷，再裁剪成目标子树。
3. 现有预取只缓存了解析后的少量节点，没有缓存 CLI 返回的原始 JSON。
4. 结果是展开热点目录时仍会出现明显等待，且等待成本会随着点击路径反复支付。

## 目标

1. 扫描会话内只跑一次目录树 CLI，后续展开和预取不再重复调用 CLI。
2. 从 CLI 得到的 JSON 必须缓存，并能按路径快速复用。
3. 目录树展开即使未命中预取，也只能走本地缓存解析路径，不能再走重扫磁盘路径。
4. 解析缓存保持小而可控，避免重新回到整棵树常驻内存。

## 方案

### 会话缓存

- 首次扫描目录树时，CLI 标准输出不直接整段读入内存，而是流式写入当前会话临时文件。
- 临时文件只在当前进程内有效；重新扫描后旧会话整体失效并删除。
- 该文件是目录树会话的唯一原始数据来源。

### 路径索引

- 在 CLI 输出落盘后，执行一次全量索引构建。
- 为每个目录记录：
  - 路径
  - 大小
  - 直接文件数
  - 总文件数
  - 总子目录数
  - 直接子目录路径列表
  - 直接 `files` 数组在临时文件中的切片位置
- 展开某个目录时，直接通过路径索引定位，不再从根节点重新递归查找。

### 节点物化

- 目录节点展开时，只做两件事：
  - 从索引中拿直接子目录摘要；
  - 从临时文件读取该目录自己的 `files` 数组切片，解析直接文件。
- 不再为展开父目录而重扫整棵子树。
- 解析后的 `StorageItem` 节点只放入小型内存缓存，控制上限。

### 预取

- 后台预取继续存在，但职责改为“预热节点物化缓存”，而不是重复触发 CLI。
- 预取未命中时，用户点击展开仍只会命中本地 JSON 会话缓存，不会感觉到磁盘扫描级别的延时。

## 影响范围

- `src/AiCleanVolume.Desktop/Services/FolderSizeRankerScanProvider.cs`
  - 改为会话级 JSON 临时文件和路径索引驱动。
- `src/AiCleanVolume.Desktop/Services/StorageTreePrefetchCoordinator.cs`
  - 继续负责后台预热，但不再承担原始数据缓存职责。
- `src/AiCleanVolume.Desktop/MainWindow.cs`
  - 展开逻辑继续优先命中预热缓存，未命中时也只走本地 JSON 会话解析。

## 验收

- 同一轮扫描中，目录树首次扫描后不再因展开节点重复调用 CLI。
- 展开热点目录时不再出现可感知的“正在加载...”常态延迟。
- 未命中预热的冷门目录也能快速展开。
- 重新扫描后旧会话不会串入新目录树。
