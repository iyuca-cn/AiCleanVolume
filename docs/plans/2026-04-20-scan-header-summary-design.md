## 背景

当前扫描页顶部是偏表单化的双行工具栏，包含盘符、路径、最小体积、层级限制、排序，以及三张信息卡。信息虽然完整，但视觉重心分散，和用户提供的“磁盘选择 + 容量摘要 + 扫描状态”样式差距较大。

## 目标

1. 将扫描页顶部改成更接近系统磁盘工具的摘要头部。
2. 保留现有高级筛选项，不折叠、不删除，但编排要更协调。
3. 将顶部摘要聚焦为当前卷信息、容量分布和扫描状态，而不是继续使用三张信息卡。
4. 将 `最小 MB` 与 `每层限制` 的默认值统一改为 `-1`，表示不限。

## 方案

### 顶部结构

- 顶部卡片改成三段结构：
  - 左侧两行筛选区：第一行仅保留 `选择:`、盘符下拉和 `扫描` 按钮；第二行放 `位置`、`最小`、`限制`、`排序`。
  - 右侧磁盘摘要区：显示当前卷、总空间、已用、可用、预留空间。
  - 底部状态区：显示扫描状态文案和绿色进度条。
- 原有“当前根路径 / 最近扫描 / 建议数量”信息卡从顶部移除。

### 交互与数据

- 盘符切换时，右侧摘要即时刷新。
- 路径输入变化时，尽量根据路径所属卷刷新右侧摘要。
- 扫描开始时，底部状态切换为“正在扫描”；完成后更新为“扫描完成 x.xx 秒”，并让进度条进入完成态。
- 若扫描失败，状态区改为错误态。

### 默认值

- `ApplicationSettings.ScanSettings` 默认值改为 `-1 / -1`。
- `ScanRequest` 默认值同步改为 `-1 / -1`。
- 桌面端界面的回退值和仓库内默认 `appsettings.json` 同步改为 `-1 / -1`。

## 影响范围

- `src/AiCleanVolume.Desktop/MainWindow.cs`
- `src/AiCleanVolume.Core/Models/ApplicationSettings.cs`
- `src/AiCleanVolume.Core/Models/CoreModels.cs`
- `src/AiCleanVolume.Desktop/appsettings.json`

## 验收

1. 顶部布局变为左侧筛选、右侧磁盘摘要、底部扫描状态三段式。
2. 扫描按钮位于扫描区顶部，而不是标题栏右侧。
3. 右侧摘要能展示总空间、已用、可用、预留空间。
4. 扫描完成后显示耗时和完成态进度条。
5. `最小 MB` 与 `每层限制` 默认显示为 `-1`，并实际按“不限”生效。
